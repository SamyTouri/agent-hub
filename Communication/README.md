# Communication workstream

## Objective

Align every public explanation of Agent Reputation with the direction validated on 21 July 2026:
Agent Reputation is an independent, cross-registry decision-support layer used **before** an agent
or a human buys a service or product from an agent.

The former democratic-governance constitution was superseded on 2026-07-23 before
any voting mechanism operated. Current founder-led operating principles are a separate layer.

## Operating-principles compatibility

No participation creates membership, governance, ownership or financial rights. The
0–100 reputation concept is not a governance parameter and must not be used as the
product's central promise or as a universal purchase verdict.

## Message hierarchy

1. **Decision first** — before buying, determine what the candidate agent claims, what it has
   actually done, which sources support it, what contradicts it, and what remains unknown.
2. **Independent advice** — the recommendation is not paid for by the seller, the marketplace or
   an investor that benefits from the transaction.
3. **Evidence dossier** — cross-registry facts, outcomes, changes, incidents, contradictions and
   missing information are kept distinct, dated and sourced.
4. **Contextual analysis** — the same agent can be suitable for one mission and unsuitable for
   another. The output is a decision memo, not a universal score.
5. **Neutral outcomes** — advice may be to proceed, choose another provider, demand guarantees,
   reduce exposure, postpone or not buy.

## Boundaries

Agent Reputation must not be presented as:

- a marketplace;
- a universal ranking or scoring authority;
- the largest directory as an end in itself;
- a guarantee that an agent is safe;
- a seller-paid reputation service.

Existing listings, ratings and imported signals remain useful inputs. They are not the conclusion
and must never be blended into a universal verdict.

## Honest status during the MVP

The evidence dossiers and pre-purchase analyses are being tested manually. Public copy must state
that status plainly and invite real decision cases without implying that a mature automated service
already exists.

## Public surfaces in scope

- Homepage and global metadata
- Human-facing explanations
- Agent-facing instructions (`llms.txt`, MCP instructions and A2A card)
- Directory, profile, tag and ranking language where it implies a universal verdict
- Repository/registry descriptions that function as public product communication

## Acceptance check

A new visitor should understand within one screen:

- the decision being helped;
- who the primary customer is;
- why the advice can be independent;
- what is delivered;
- what Agent Reputation does **not** claim to know.
